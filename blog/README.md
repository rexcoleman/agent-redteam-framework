# blog/ — Content Artifacts for Agent Red-Team Framework

This directory contains all content artifacts derived from the Agent Security Red-Team Framework research.

## Files

| File | Purpose | Status |
|------|---------|--------|
| `draft.md` | Technical blog post (primary content artifact) | DRAFTED |
| `conference_abstract.md` | BSides / DEF CON AI Village CFP abstract (250 words) | DRAFTED |
| `linkedin_post.md` | LinkedIn native post (~2,000 chars, link in first comment) | DRAFTED |
| `substack_intro.md` | Substack newsletter intro with personal angle | DRAFTED |
| `images/` | Figures for blog post (generated by scripts, not manual) | GENERATED |

## Content Pipeline

1. **Technical blog post** (`draft.md`) is the canonical source of truth
2. All other formats adapt from the blog post — not the other way around
3. Figures are generated by `scripts/make_report_figures.py` and copied to `blog/images/`
4. Distribution plan and Gate 9 checklist are in `docs/CONTENT_PLAN.md`

## Image Sources

All images in `images/` are generated from raw experiment data:
- `attack_success_rates.png` — per-class success rates with error bars (3 seeds)
- `defense_comparison.png` — 4-layer defense effectiveness comparison
- `controllability_analysis.png` — observability vs attack success correlation
- `defense_effectiveness.png` — defense reduction by attack class
- `attack_by_class.png` — attack class heatmap
- Mermaid source files (`.mmd`) for architecture diagrams
